バーコミュニティアプリ：開発要件定義書
📋 プロジェクト概要
プロジェクト名
VCバーコミュニティアプリ（仮称）

プロジェクトの目的
学生起業家とベンチャーキャピタリストが集まるバー型コミュニティスペースにおいて、「誰が今ここにいるか」をリアルタイムで可視化し、偶然の出会いとマッチングを促進するWebアプリケーションの開発。

**開発方針**: 初期フェーズではWeb UIで開発し、迅速な実証実験を実施。将来的にReact Nativeでモバイルアプリ化を検討。

ターゲットユーザー
メインユーザー: 学生起業家（20-25歳）
サブユーザー: ベンチャーキャピタリスト、メンター、起業家
想定ユーザー数: 初期50-100名 → 6ヶ月で300-500名
開発スコープ
Phase 1: MVP（実証実験版） ← 今回の開発対象
予算：20万円
期間：1ヶ月以内
目的：コンセプト検証と初回イベントでの使用

🎯 コアバリュープロポジション
解決する課題
学生が「VCに会いたいけど、いつどこにいるかわからない」
VCが「優秀な学生と出会いたいけど、効率的な方法がない」
物理的な場所はあるが、「誰がいるか」の情報の非対称性
提供する価値
✅ リアルタイムで在館者を可視化
✅ 「会いたい人がいる時」に訪問を決められる
✅ プロフィール情報で話しかけるきっかけを提供
✅ オンライン・オフライン融合による自然なマッチング

💻 技術スタック（採用）
フロントエンド
**採用**: Next.js 14 (App Router) + TypeScript
**スタイリング**: Tailwind CSS
**理由**:
- Web UIで迅速な開発・テスト可能
- Vercelによる即座のデプロイ
- App Store/Google Play登録不要（コスト削減）
- 実証実験後にReact Nativeでモバイルアプリ化を検討

バックエンド・インフラ
**採用**: Firebase
- Authentication（認証）
- Firestore（NoSQLデータベース）
- Realtime Database（リアルタイム在館者管理）
- Cloud Storage（画像保存）
**デプロイ**: Vercel（無料枠使用）
**理由**: サーバーレス、迅速な開発、低コスト運用

🔒 セキュリティ・プライバシー要件
必須要件
✅ パスワードのハッシュ化（Firebase Authで自動対応）
✅ HTTPS通信の徹底
✅ ユーザーデータへのアクセス制御（Firestore Security Rules）
✅ プロフィール公開範囲の選択肢提供
✅ 位置情報は取得しない（チェックインは手動操作）
セキュリティルール例（Firestore）
Copyrules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }
    match /presence/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }
  }
}

📊 成功指標（KPI）
実証実験で測定する指標
チェックイン率: 来館者の何%がアプリを使用したか → 目標：70%以上
アクティブユーザー率: 1イベントあたりのアプリ利用者数 → 目標：20名以上
平均滞在時間: アプリで記録された滞在時間 → 目標：3時間以上
プロフィール閲覧数: 1ユーザーあたりの平均閲覧数 → 目標：5プロフィール/回
ユーザー満足度: イベント後アンケート → 目標：NPS 50以上
